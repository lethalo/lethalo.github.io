0x01 special number

给出了一段代码，$b==$lock，php弱类型，0==”string”,根据$pattern = '/^(?=.*[0-9].*)(?=.*[a-zA-Z].*).{7,}$/ ';构建payload

\ 一般用于转义字符

^ 断言目标的开始位置(或在多行模式下是行首)

$ 断言目标的结束位置(或在多行模式下是行尾)

. 匹配除换行符外的任何字符(默认)

[ 开始字符类定义

] 结束字符类定义

| 开始一个可选分支

( 子组的开始标记

) 子组的结束标记

? 作为量词，表示 0 次或 1 次匹配。位于量词后面用于改变量词的贪婪特性。 (查阅量词)

  • 量词，0 次或多次匹配
  • 量词，1 次或多次匹配

    { 自定义量词开始标记

    } 自定义量词结束标记

0x02 sc2_player

ida看下，题目中将flag进行一系列操作后进行判断,先改下函数名

先检查flag长度,是不是28位

看下第一个函数encrypt_0，sec[]=(i+35)^fir[i+7*n]

第二个函数encrypt_1,fir[]=fir[i+7*n]^0x34

第三个函数encrypt_2,sec[]=n^(i+7*n)^flag[i+7*n]

最后一个check函数

解密脚本

baby_crack

和前一题差不多，都是对输入的flag进行操作，然后加密后进行比较

第一个函数是进行一些位运算，4位一组

第二个函数是进行简单的位交换，

第三个函数是进行映射，对应到一个256位数组的值

最后一个函数的功能和strcmp相同

得到映射和交换之前的字符串
